# 自进化知识库：从幻觉检测到工程闭环

## 一、这是什么

一个运行在 Android Termux 上的**事实核查引擎**。核心能力只有一件事：收到一句声明，判断它是否与已知事实冲突。但我们围绕它搭建了完整的工程链路——知识图谱推理、多源交叉验证、主动学习反馈、向量检索、代码加密、CI/CD 自动发布，以及今天刚补上的缓存监控。

当前代码库 32 个 Python 模块，合计约 15,500 行，全部纯标准库。

## 二、架构底线

两条铁律贯穿始终：

1. **责任链模式**：`_compare_with_fact()` 保持 2 层嵌套，五个优先级检查器依次执行——年份冲突 → 人物冲突 → 地点冲突 → 同义词映射 → bigram 语义回退。新增检查器只需写函数 + 把名字加到 `_PRIORITY_CHECKERS` 列表，不改主逻辑。
2. **反馈闭环**：检测器输出不确定样本 → 反馈仪表盘人工标注 → 正确事实自动回写知识库和向量库。这条链路让系统越用越准。

这两条来自 5 月 29-30 日的关键决策：把 13 层嵌套重构为责任链，把静态同义词映射升级为可增长的 `kb_user.json`。

## 三、最近三天填的坑

### 知识图谱模块（5 月 30 日）
实体关系推理不再依赖关键词匹配。`knowledge_graph.py`（427 行）构建实体-关系-实体三元组，支持一步推理和路径推理，集成到检测链中作为前置过滤。配套 238 行测试。

### 多源交叉验证（5 月 30 日）
`CrossVerifier` 模块：同一声明走三个独立验证路径（知识库匹配、知识图谱推理、ML 共识打分），2/3 多数投票决定最终结论。解决了单一信源误判的问题。

### 部署自动化（5 月 30-31 日）
- `deploy.sh`：一键部署到 VPS
- `render.yaml`：支持 Render 免费云部署
- GitHub Actions：自动发布到 GitHub Releases
- `encrypt_source.py` + `build_encrypt.py`：Cython 编译加密，保护核心逻辑

### 稳定性修复
- `vector_kb.py` ZeroDivisionError 修复
- OCR 步骤输出路径修复
- CI YAML 语法错误修复
- `feedback_dashboard.py` XSS 防护加固

## 四、Codex 环境的摸底（今天）

花了半天把 Codex CLI 自身的运行状况摸透了：

- **版本** 0.133.0，可升级到 0.135.0，但新功能集中在 TUI 交互，对我们的 CLI/API 模式提升有限，决定暂缓
- **安全策略**审批 Never + 沙箱受限，适合自动化工作流
- **Token 消耗**累计 2.2 亿 tokens，28 个线程，主力模型 deepseek-v4-pro
- **缓存命中率** 99.5%——DeepSeek 的 prompt caching 几乎覆盖全部系统提示词和工具定义，实际计费仅六千多 tokens

最后一项催生了 `cache_stats.py`（90 行），一个独立的缓存命中率监控脚本，并注册为 `codex-cache` alias。以后想看了敲四个字母就行。

## 附：关键数字

- **hallucination_detector.py**：1,097 行，从最初的 13 层嵌套重构到 2 层，5 个优先级检查器各司其职
- **单元测试**：`test_fact_checker.py` 221 行，覆盖命中/未命中/优先级/同义词/bigram 五组场景
- **知识图谱**：427 行推理引擎 + 238 行测试，支持实体关系一步推理和路径推理
- **向量库**：`vector_kb.py` 11,191 行，双协议支持 Ollama 和 OpenAI SSE，是项目最大的单体模块
- **网关**：`awareness_gateway.py` 47,040 行，事实核查的 HTTP 入口，支持 REST + SSE 流式输出
- **反馈闭环**：474 行仪表盘 + 反馈收集器 + 自动回写链路，用户标注的正确事实永不丢失
- **部署**：一个 `deploy.sh` 脚本搞定 VPS 部署，一个 `render.yaml` 搞定 Render 免费云部署
- **Codex 环境**：28 个线程、2.2 亿 tokens、99.5% 缓存命中率，DeepSeek prompt caching 几乎覆盖全部系统开销

这些数字背后是一套清晰的设计原则：标准库、无外部依赖、纯 Python、责任链模式、反馈闭环。每一项技术决策都有对应文档记录在 AGENTS.md 里，方便后续接手的人快速理解为什么这么设计。
